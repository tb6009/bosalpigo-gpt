### 📉 핵심 트렌드: 재미 ↓ 규칙 ↑

```
v1.0  재미████████ 규칙██               재미:규칙 = 4:1
v1.1  재미█ 규칙██████                   재미:규칙 = 1:6
v1.2  재미█ 규칙█████████                재미:규칙 = 1:9

```

**결론**: 재미가 v1.0에서 v1.3으로 오면서 **87.5% 감소**, 규칙은 **700% 증가**.

## 🎭 버전별 캐릭터 DNA

### v1.0 — "시니컬한 코미디언 친구"
```
프롬프트 특징:
- 캐릭터 = 마음 혼자 (모미 없음)
- "시니컬한 코미디언", "재밌는 게 기본값"
- "비유, 드립, 셀프디스 OK"
- "딱딱한 교과서 톤 절대 금지"
- 규칙은 2개뿐: "정확하게" + "공감 먼저"
- MI 없음 — 자유로운 대화
- 내장 지식: 게으름 7경로, 식사, 봄철, 감정 (직접 시스템에 박혀있음)

분위기: 동네 술집에서 과학 덕후 친구랑 수다
```

### v1.1 — "듀오 에이전트 탄생, MI 도입"
```
프롬프트 특징:
- 모미(몸) + 마음(심리) 분리
- MI 정신·OARS·DARN-CAT 도입
- 오프닝 프로토콜 정형화
- 등장 규칙 (몸→모미, 마음→마음)
- 온보딩 모달 (폼 형식)
- 재미 키워드 급감 (8→1)

분위기: 대학 상담센터의 쿨한 상담사 듀오
```

### v1.2 — "대화형 온보딩으로 전환"
```
프롬프트 특징:
- 폼 온보딩 제거 → 대화로 수집
- "갱년기이세요?" 같은 분류 라벨 금지 추가
- 프로필 메타태그 [[PROFILE:...]] 파싱
- 규칙이 더 많아짐 (6→9)
- 톤 지시는 그대로

분위기: 규칙 잘 지키는 상담사 듀오
```

### v1.3 — "주제 경계 + 이모티콘 규칙 강화"
```
프롬프트 특징:
- 재정·진로·관계 등 6개 금지 주제 추가
- 이모티콘 "절대 규칙 (어기면 실격)" 섹션
- fixCharFaces() 사후 교정 코드
- 규칙 16개 — 프롬프트의 다수가 "하지 마" 목록
- 재미 관련 지시: "시니컬하되" 1회뿐

분위기: 규칙집 들고 온 진지한 상담사
```
