## 🎯 측정하려는 4개 핵심 결과

### 1. 자책감 감소 (Self-blame Reduction)

**왜 중요한가**
마음·모미의 핵심 미션은 "자기를 덜 탓하게 돕는 것". 이 결과가 안 나오면 다른 지표 다 무의미.

**측정 방법**

| 층위 | 방법 | 빈도 |
|------|------|------|
| **즉시** | 세션 전·후 자책감 척도 1~10 슬라이더 (옵션) | 매 세션 |
| **대화 내** | LLM-as-judge 가 대화 읽고 "자책 표현 → 정상화 표현" 전환 횟수 카운트 | 샘플 평가 |
| **지연** | 1주 뒤 follow-up: "지난번 얘기한 거 떠올리면 기분 어때요?" | 월 1회 |

**자책 표현 키워드 (자동 감지)**
- "내가 왜 이럴까", "나만 이상해", "게으른 건가", "한심해", "망했어"
- 대화 전반부 등장 → 후반부 감소하면 긍정 지표

**정상화 표현 (에이전트가 제공해야 할 것)**
- "이건 {OO님}만 그런 거 아니에요"
- "몸이/뇌가 그렇게 반응하는 게 정상이에요"
- "{구체적 메커니즘} 이라 일어나는 일이에요"

**목표값**: 세션 내 자책 표현 ≥2회 → 종료 시 정상화 수용 ≥1회

---

### 2. 도움됨 (Perceived Helpfulness)

**측정 방법**

| 질문 | 척도 | 시점 |
|------|------|------|
| "오늘 대화 도움이 됐나요?" | 1~5 별점 | 세션 종료 |
| "어떤 부분이 가장 도움됐어요?" | 자유 텍스트 (선택) | 세션 종료 |
| "다시 이런 얘기 나눌 수 있을 것 같아요?" | Y/N | 세션 종료 |
| NPS: "마음·모미를 친구에게 추천하시겠어요?" | 0~10 | 월 1회 |

**수집 방식**
- 사용자 자발적 답변 (강요 X)
- "지금 답하기 싫어요" 선택지 항상 존재
- 답변 → 브라우저 localStorage만 저장 (익명 메타데이터만 기증 버튼 눌렀을 때 서버 전송)

**목표값**
- 도움됨 ≥ 4.0/5.0 (평균)
- NPS ≥ 30 (양호 기준)

### 3. 재방문 (Retention)

**측정 방법 — 서버 저장 없이 localStorage 기반**

```javascript
maeum.daily = {
  first_visit: "2026-04-13",
  last_visit: "2026-04-17",
  total_sessions: 7,
  streak_days: 4,
  gap_days: [0, 2, 1, 0, 1],    // 방문 간격
}
```

**핵심 지표**
- **D1 retention**: 첫 방문 다음날 재방문율
- **D7 retention**: 7일 내 재방문율
- **평균 세션 간격** (days)
- **연속 접속 일수 (streak)**

**추적 안 하는 경우 대안**
- 사용자 자발적 "지난번 대화 이어가기" 버튼 클릭 = 재방문 의지 지표
- 설문: "한 달 뒤에도 쓸 것 같나요?"

**목표값** (테스터 기준)
- D1 ≥ 30%
- D7 ≥ 50%
- 주당 평균 1회 이상 = 건강한 사용

### 4. 작은 실천 실행률 (Action Follow-through)

**왜 중요한가**
MI 2순위 원칙 = 행동 변화. 대화 끝에 "오늘 뭐 해볼까요" 약속이 **실제 실행됐는가**가 진짜 지표.

**측정 방법**

| 시점 | 행동 |
|------|------|
| **세션 종료** | "오늘 뭐 하나만 해볼까요?" 약속 받기 → 저장 |
| **다음 세션 시작** | "지난번 약속한 거 어떻게 됐어요?" 질문 |
| **답변 분류** | 실행함 / 부분 실행 / 못함 / 잊음 |

**localStorage 구조**
```javascript
maeum.promises = [
  {
    date: "2026-04-17",
    promise: "오늘 밤 10분 스트레칭",
    specificity: "high",    // 언제·얼마나·어떻게 명시 여부
    confidence_0_10: 7,     // 척도 질문 결과
    outcome: null,          // 다음 세션에서 업데이트
  }
]
```

**자동 분석**
- 약속 구체성 (언제·얼마나·어떻게 모두 포함?) → 실행률과 상관관계 분석
- 척도 7점 이상 → 실행률 비교
- 실행 안 됐을 때 "왜?" 대화로 다시 유발

**목표값**
- 구체적 약속 비율 ≥ 70%
- 실행 or 부분 실행 ≥ 60%

## 🛠 측정 시스템 — 단계별 실행

### Phase 1 (바로) — 자동 기계 체크
- `eval/run_mechanical.js` 작성
- 시나리오 5개에 대해 API 호출 → 포맷·금지어 자동 검증
- 배포 전 필수 관문

### Phase 2 (1~2주) — LLM judge
- MITI 평가 프롬프트 완성
- Claude Opus 4.6로 대화 평가
- 시나리오 20개 확장

### Phase 3 (월) — 사용자 피드백
- 세션 종료 시 별점·한 줄 피드백 (옵션)
- 익명 기증 버튼
- 자책감 슬라이더 (선택)

### Phase 4 (분기) — 장기 결과
- 1주·1달 follow-up 메시지 (옵션)
- 실천 약속 완료율 대시보드
- NPS 월간 집계
