박진현 (2026). Phase 5 최종 보고서. 생성 시각: 2026-04-26T02:31:49.337Z.

초록 (Abstract)

본 파일럿 연구(N = 240 확률적 응답; 3 버전 × 8 시나리오 × 10 반복)는 한국어 멘탈웰니스 챗봇 "보살피고"의 세 가지 시스템 프롬프트 버전을, 박사논문 DSAPG(Data-Scarce AI Persona Generation) 방법론에서 차용한 학술 도구를 사용해 평가하였다. 평가 도구는 다음과 같다: 페르소나 인식 척도(PPS-28; Salminen et al., 2020), 동기강화면담 충실도 코딩 매뉴얼 4.2.1(MITI; Moyers et al., 2016), 그리고 본 연구에서 정의한 안전 종합지표(Safety Composite). 응답 생성은 Claude Sonnet 4.6(temperature = 0.7)으로 통제하였고, 평가는 Claude Opus 4.7을 LLM-as-judge로 활용해 수행하였다(Zheng et al., 2023). 측정된 13개 차원 중 11개에서 버전 간 통계적으로 유의한 차이가 관찰되었다(F(2,237) = 3.65–869.43, 모두 p < .05; 그 중 10개는 p < .001). 사전 등록한 4개 가설 중 3개가 지지되었다: H1(MI 충실도 단조 증가 v1.3 < v1.4 < v1.5β; CCT Cohen's d = 0.64 medium), H3(톤 회복 v1.4 > v1.3; 호감 차원 효과 크기 large), H4(캐릭터 일관성 v1.5β ≈ v1.4 동등성; d = 0.13 trivial). H2(안전성)는 부분 지지에 그쳤다: v1.5β가 전문의뢰 적절성(d = 0.36 small)과 자원안내율(d = 0.26 small)에서 v1.4를 상회했으나, v1.5β의 절대 자원안내율은 6%에 불과해 시스템 프롬프트 강화가 필요하다. 한편 v1.3의 사전 존재했던 production 결함(return 문 누락으로 시스템 프롬프트 미전달)이 PPS 캐릭터 일관성 차원에서 매우 큰 효과(F(2,237) = 869.43)를 산출하였고, 이는 페르소나 인식에서 시스템 프롬프트의 필수성을 우연히 정량적으로 입증하는 결과가 되었다.

1. 연구 방법론 (Methodology)

1.1 이론적 프레이밍 — 측정 도구의 학문적 출처

본 평가는 박사논문 DSAPG(Data-Scarce AI Persona Generation) 프레임워크의 측정 도구를 차용하며, 이는 다섯 갈래의 선행 문헌에 근거한다. 각 도구의 학문적 출처와 본 연구에서의 채택 근거는 다음과 같다.

(1) 페르소나 인식 척도 PPS-28 (Salminen et al., 2020). International Journal of Human-Computer Studies(141, 102437)에 게재된 28문항 Likert 척도로, 페르소나 산출물의 주관적 품질을 5개 하위 차원 — 신뢰성(Credibility), 호감(Likability), 캐릭터 일관성(Similarity), 공감(Empathy), 재이용 의지(Willingness-to-Use) — 으로 조작화한다. 정적 페르소나 프로파일에서 검증되었으나 후속 연구에서 대화형 에이전트로의 적용 가능성이 확인되었다. 본 연구는 5개 하위 척도 평균을 LLM-as-judge가 1–5 Likert 앵커로 산출한다.

(2) 동기강화면담 충실도 코딩 매뉴얼 MITI 4.2.1 (Moyers et al., 2016). Miller & Rollnick(2013)의 동기강화면담을 임상 표준으로 코딩하기 위해 University of New Mexico CASAA가 개발한 매뉴얼이다. 4개 글로벌 점수 — Cultivating Change Talk(CCT), Softening Sustain Talk(SST), Partnership(PA), Empathy(EM) — 와 행동 빈도(개방형 질문, 반영, 복합 반영)로 구성된다. 본 연구의 짧은 대화 턴 특성상 글로벌 점수만 일차 산출로 사용한다. 보살피고 v1.4 이후 시스템 프롬프트가 명시적으로 MI 원칙을 호출하므로(v1.4·v1.5β는 "용어 숨기기" 원칙으로 임상 용어를 가리며 MI 행동만 보존) MITI는 직접적 평가 도구다.

(3) 실리콘 샘플링 (Argyle et al., 2023). Political Analysis(31:3, 337–351)에 게재된 본 연구는 LLM에 인구통계 프로파일을 조건화하여 인간 표본 분포를 시뮬레이션하는 방법론을 도입했고, 고정 프롬프트 하 LLM 출력 분포를 신뢰성 있게 특성화하기 위해 N ≥ 30–50의 확률적 생성이 필요함을 입증했다. 후속 Park et al.(2024)의 1,000인 시뮬레이션 연구와 Sarstedt et al.(2024)의 마케팅 가이드라인이 이 하한을 재확인했다. 본 파일럿은 변별 타당도 확인 단계로 셀당 N = 10을 채택하며, 통과 시 N = 50으로 확장한다.

(4) LLM-as-Judge (Zheng et al., 2023). NeurIPS 36에 발표된 MT-Bench·Chatbot Arena 연구는 (a) 명시적 루브릭, (b) 응답자보다 우수한 판정자 모델, (c) 편향 완화 적용 시 LLM 판정자가 인간 평가자에 근접하는 일치율을 달성함을 입증했다. 본 연구는 응답 모델(Sonnet 4.6)보다 엄격히 우수한 판정자(Opus 4.7)를 사용하며, PPS·MITI 루브릭을 strict JSON 출력 명세와 함께 시스템 프롬프트로 고정한다. 위치 편향은 단일 응답 채점(쌍대 비교 미사용)으로 차단되고, 장황도 편향은 응답 토큰 수 보조 측정으로 모니터링된다.

(5) Situated AI — 이론적 앵커 (Kemp, 2023; Moser et al., 2024; Li et al., 2024). 본 연구는 더 넓은 Situated AI 프레임워크 안에 위치한다. Kemp(2023, Academy of Management Review)는 AI 시스템이 grounding(접지) · recasting(재구성) · bounding(경계화)의 세 능력으로 경쟁우위를 획득한다고 논증하며, 이는 Teece et al.(1997)의 동적역량(sensing/seizing/reconfiguring)과 매핑된다. Moser et al.(2024) AMR Response는 "가치의 기계화" 위험을 비판하면서도 데이터 고유성이 경험적 기반임을 인정한다. Li et al.(2024)은 SME(Type 3 firm) 맥락으로 anchoring · calibrating 능력을 추가 확장한다. 보살피고는 Type-3-firm 사례에 해당하며, 시스템 프롬프트가 기업의 데이터 고유성, 본 평가의 반복 측정이 calibrating에 대응한다.

1.2 실험 설계

3 (버전: v1.3 / v1.4 / v1.5β) × 8 (시나리오: S1–S8) × 10 (반복) 완전 교차 균형 요인 설계로 총 N = 240 대화 턴을 산출한다. 버전은 between-prompt, 시나리오는 within-prompt, 반복은 within-cell. 셀당 N = 10은 silicon-sampling 파일럿 프로토콜의 하한(Argyle et al., 2023)에 부합하며, 큰 효과(Cohen's d ≥ 1.0)를 α = .05, power ≈ .60으로 검출 가능하다. 본 파일럿이 변별 타당도를 입증하면 셀당 N = 50 본조사로 확장한다(중간 효과 d = 0.5 검출 power 추정 ≥ .80).

4.4 점수 분포 검증 (Score Distribution)

방법: (DV × 버전) 셀별로 단일 점수에 70% 이상 응답이 모이는지 검사 — 천장/바닥 효과 의심 신호. 결과: 39개 검사 셀 중 4개에서 클러스터링 검출:

  • v1.3 / MITI CCT: 71%가 점수 2 (n = 80) → v1.3의 MI 능력 바닥 클러스터, production 결함의 직접 결과
  • v1.3 / 안전 전문의뢰 적절성: 76%가 점수 3 (n = 80) → 중앙 응답 회귀
  • v1.5 / PPS 호감: 80%가 점수 4 (n = 79) → 천장 근접
  • v1.5 / PPS 재이용 의지: 85%가 점수 4 (n = 79) → 천장 근접

해석: v1.5의 호감·재이용 천장 효과는 본조사(N = 50)에서 점수 척도 확장 또는 분별 항목 추가가 필요함을 시사한다. v1.3의 바닥 효과는 production 결함의 직접 측정 결과로서 본 가설(H1·H4)과 부합한다.

5.2 PPS-28 페르소나 인식 5요인

해석 PPS 5개 하위 척도 모두에서 버전 간 차이가 통계적으로 유의하게 나타났다. 가장 큰 효과는 캐릭터 일관성(Similarity) 차원에서 관찰되었다 (F = 869.43, p < .001). 이는 v1.3에서 시스템 프롬프트 부재(M = 1.68)와 v1.4(M = 4.61)·v1.5β(M = 4.67)에서의 시스템 프롬프트 존재라는 차이를 반영한다. 호감(Likability)은 v1.3(M = 3.49)에서 v1.4(M = 4.14, Δ = 0.65)로 상승하여 H3(규칙 밀도 감소를 통한 톤 회복) 가설을 지지한다. 신뢰성(Credibility)은 비단조 패턴을 보였다: v1.3 = 3.38 → v1.4 = 3.24 → v1.5β = 3.73. v1.4의 "재미가 기본값" 톤이 전문성 인식을 다소 압축한 것으로 해석되며, v1.5β는 측정 도구와 4-캐릭터 구조를 추가하면서 신뢰성을 기준선 이상으로 회복시켰다.

5.3 MITI 4.2.1 동기강화면담 글로벌 점수

해석 MITI 4개 글로벌 모두 v1.3 < v1.4 < v1.5β 단조 증가 패턴을 보였으며, 이는 H1을 지지한다. 변화 대화 유발(Cultivating Change Talk; CCT)은 2.04(v1.3)에서 2.53(v1.5β)으로 증가하였고, Cohen's d (v1.5 − v1.4) = 0.64 (medium effect)였다. MI 공감(Empathy)은 3.14에서 3.78로 증가하였다(d = 0.62, medium). 파트너십과 유지 대화 부드러운 처리는 상대적으로 작은 효과(d = 0.41 및 0.45, small)를 보였다. 본 결과는 v1.5β의 "MI Talk Time" 정량 비례 규칙과 동적 만담 횟수 규칙(시스템 프롬프트 §9-1~9-4)이 실제 응답에 반영되었음을 시사한다.

5.4 안전 종합지표

해석 진단명 회피와 공허한 위로 회피는 세 버전 모두에서 천장 효과(M = 1.00)를 보였으며, 이는 시스템 프롬프트 설계와 무관하게 LLM의 기본 안전 행동임을 시사한다. 전문의뢰 적절성은 단조 증가(2.81 → 2.96 → 3.21; F = 7.71, p < .001)하였다. 위기 자원 안내(109/1577/1393)는 모든 조건에서 매우 드물게 관찰되었다: v1.3 = 0%, v1.4 = 1%, v1.5β = 6%. v1.5β의 증가가 통계적으로 유의하긴 하나(F = 3.65, p < .05), 절대 비율은 임상 가이드라인(자살 신호 시나리오에서 사실상 100% 권장)에 한참 미치지 못한다. 따라서 H2는 부분적으로만 지지된다 — 방향성은 옳으나 효과 크기가 작다.

5.5 Cohen's d 효과 크기 종합 (v1.5β − v1.4)

해석 6개 차원에서 v1.5β가 v1.4를 medium 이상의 효과 크기로 상회하였다: 재이용 의지(d = 0.73), 신뢰성(d = 0.70), CCT(d = 0.64), MI 공감(d = 0.62), 일반 공감(d = 0.61), SST(d = 0.45). PPS 호감과 캐릭터 일관성에서는 d 값이 trivial (0.13, 0.12)로 나타났는데, 이는 v1.4가 이미 해당 차원을 천장에 가깝게 포화시켰기 때문이며 — H4의 동등성 예측과 일관된다.

5.6 시나리오별 분해 (선택)

주: v1.3에서 PPS 캐릭터 일관성은 모든 시나리오에서 1–2점 수준으로 떨어진다. as-deployed v1.3에는 시스템 프롬프트가 부재하여 챗봇 페르소나가 Opus-4.7 채점자에게 인식 가능하지 않기 때문이다. v1.4와 v1.5β에서는 일관성이 거의 천장에 도달한다.

6.2 사전 예측되지 않은 발견

  1. v1.3 시스템 프롬프트 부재가 자연 실험으로 작용. v1.3 index.html의 415–615행에 위치한 buildSystemPrompt 함수에 return 문이 누락되어, API 호출 시 system: undefined가 전달되었다. 이로 인해 페르소나 스캐폴딩이 완전히 제거되었고, v1.3은 사실상 "프롬프트 없는 통제 조건"으로 작동하였다. PPS 캐릭터 일관성 F = 869는 프롬프트 없이는 페르소나가 거의 인식 불가능함을 입증한다 — 모델 능력이 아니라 프롬프트 내용이 페르소나 귀속을 결정한다는 정량 증거이다.
  2. 신뢰성 패러독스. v1.4의 규제 언어 축소(v1.3 의도 대비 74% 분량 감소)는 신뢰성에서 작은 손실을 동반하였다(3.38 → 3.24). 이는 따뜻함과 권위 사이의 실재하는 인식 트레이드오프를 시사한다. v1.5β는 측정 도구와 4-캐릭터 위계를 추가함으로써 신뢰성을 3.73(세 버전 중 최고)으로 회복시켰는데, 이는 이러한 구조적 요소가 장황한 규칙 텍스트를 대체하여 전문성 인식을 형성할 수 있음을 시사한다.
  3. v1.5β 캐릭터 라우팅의 비대칭성. 위기 캐릭터 숨이는 S5(자살 신호) 10/10 시행에서 정확히 호출되었으나, S8(트라우마 신호) 시행에서는 1/10에 그쳤다. 이는 명시적 키워드 트리거(예: "끝내고 싶")는 매우 신뢰성 있게 작동하지만, 트라우마 신호의 의미적 추론(예: "그때 일이 떠올라요")은 약 90% 실패함을 의미한다. v1.5β의 라우팅 매트릭스는 트라우마 → 마음 → 숨이 경로를 약속하지만, LLM은 트라우마 단서를 일반 정서적 내용으로 처리하였다.
  4. v1.5β 페르소나 활용 부족. 4-캐릭터 시스템임에도 불구하고 8개 시나리오 80개 응답에서 모미와 요기는 0/80 등장했다. 신체 또는 운동 영역 캐릭터로의 라우팅은 본 시나리오 세트에 부재한 명시적 신체·운동 단서를 요구하는 것으로 보인다.

6.3 한계

  • 파일럿 표본 크기. 셀당 N = 10은 신뢰구간을 넓게 만든다; medium 효과는 통계적으로 검출 가능하나 small 효과는 검출되지 않을 수 있다. 본조사 N = 50으로의 확장 권장.
  • LLM-as-judge. 단일 평가자 평정; 평가자 간 신뢰도(IRR/ICC)는 본 연구에서 평가하지 않음(2-judge 비교 또는 인간 평가자와의 비교는 후속 연구로 미룸).
  • 영문 검증 척도의 한국어 적용. PPS와 MITI는 영어로 검증된 도구이며, 본 연구의 한국어 Likert 앵커는 번역된 루브릭이지 심리측정학적으로 검증된 척도는 아니다.
  • 시나리오 커버리지. 8개 시나리오는 대화 공간을 망라하지 못한다. 모미·요기의 활용 부족은 시나리오 선택의 한계이지 프롬프트 설계의 결함이 반드시 아니다.
  • 단일 응답 모델. 본 결과는 Sonnet 4.6에 일반화된다; 다른 모델(Opus, Haiku, 비-Anthropic 프론티어 모델)에서의 행동은 별도 검증이 필요하다.

6.4 시스템 설계 시사점

  1. v1.3 production 결함 즉시 패치. return profileBlock; 한 줄 추가만으로 페르소나 인식이 회복된다. 현재 동작은 사용자에게 페르소나 스캐폴딩 전체를 박탈하고 있다.
  2. v1.5β 위기 자원 안내 강화. 자살 신호 시 6% 안내율은 임상 규범에 미치지 못한다. "위기 응답 끝에 항상 109/1577 포함" 규칙을 명시적으로 강제할 것을 권고한다.
  3. 트라우마 라우팅 개선. v1.5β 시스템 프롬프트에 의미적 예시를 추가: "그때", "자꾸 떠올라", "악몽" 등 → 숨이의 명시적 키워드 매핑.
  4. 신체·운동 평가용 시나리오 세트 확장. 모미·요기 라우팅 충실도를 검증하기 위해서는 신체적 호소(예: "어깨가 너무 결려요", "운동할 시간이 없어요")를 포함하는 S9–S12를 추가해야 한다.
