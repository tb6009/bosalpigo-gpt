## 0. 한 줄 요약

> **MITI는 임상 면담자가 동기강화면담(MI) 정신을 얼마나 충실히 따르는지 정량 평가하는 표준 도구다. 4개의 글로벌 점수(1–5)와 9개 행동 카운트로 구성된다.**

본 평가에서는 **글로벌 4점만** 사용 (대화 턴이 짧아 행동 카운트는 변별력 약함).

## 1. MI(동기강화면담) 자체부터 이해하기

### 1-1. MI가 뭔가?

**Motivational Interviewing (MI)**: 행동 변화를 원하지만 양가감정(ambivalence)을 가진 사람들과 대화하는 방법론. 1983년 William Miller가 알코올 중독 치료에서 시작 → 현재 흡연·식이·우울·자해 등 모든 행동 변화 영역에 적용.

**핵심 정신 4가지** (Miller & Rollnick 2013):
1. **Partnership (협력)** — 전문가가 일방적으로 가르치는 게 아니라 동료처럼 함께 탐색
2. **Acceptance (수용)** — 사용자의 자율성·가치·강점 인정
3. **Compassion (연민)** — 사용자의 안녕을 우선
4. **Evocation (이끌어내기)** — 답을 주는 게 아니라 사용자에게서 끌어냄

### 1-2. MI의 핵심 기법

#### **OARS** (기본 4기법)
- **O**pen Question — 열린 질문 ("어떻게 시작됐어요?")
- **A**ffirmation — 인정·강점 짚기 ("스스로 챙기시는 게 보여요")
- **R**eflection — 반영 ("힘드셨겠어요" / 사용자 말 되비추기)
- **S**ummary — 요약·정리

#### **DARN-CAT** (변화 대화 단계)
- **D**esire — 원함 ("자고 싶어요")
- **A**bility — 능력 ("할 수는 있을 것 같아요")
- **R**eason — 이유 ("이렇게 살면 안 되니까")
- **N**eed — 필요 ("당장 바꿔야 해요")
- **C**ommitment — 약속 ("할게요")
- **A**ctivation — 활성화 ("준비 됐어요")
- **T**aking step — 실행 ("어제 시작했어요")

DARN은 변화 의지의 **씨앗**, CAT은 변화 행동의 **결과**.

#### **EPE 흐름** (정보 제공 시)
- **E**licit — 이끌어내기 ("이미 알고 계신 게 있어요?")
- **P**rovide — 제공하기 (짧게 정보)
- **E**licit — 다시 이끌어내기 ("어떻게 와닿아요?")

#### **교정반사 (Righting Reflex)**
사람들의 문제를 "고쳐주려는 본능적 반응". MI에서는 **금지**.
- ❌ "운동하셔야 해요" (지시)
- ❌ "긍정적으로 생각해 보세요" (조언)
- ❌ "왜 안 하세요?" (추궁)

→ 보살피고 시스템 프롬프트에 "교정반사 금지", "설교 금지" 명시.

## 2. MITI는 왜 만들어졌나?

### 2-1. 문제 의식

MI 트레이닝을 받은 임상가들도 **실제로 MI 정신대로 면담하는지 검증할 방법이 없었다**. 자기 보고는 부정확하고, 슈퍼비전은 시간이 많이 든다. → 객관적·정량적 평가 도구 필요.

### 2-2. MITI의 역사

| 버전 | 연도 | 변화 |
|---|---|---|
| MITI 1.0 | 2003 | 초판 |
| MITI 2.0 | 2005 | 확장 |
| MITI 3.0 | 2007 | Global 5점 도입 |
| MITI 3.1.1 | 2010 | 미세 조정 |
| **MITI 4.2.1** | **2014/2016** | **현재 표준** — Global 축소(7→4), 행동 카운트 정비 |

### 2-3. 활용 영역

- 임상 슈퍼비전 (학생 임상가 평가)
- 연구 (RCT에서 면담 충실도 보장)
- 코칭 트레이닝 효과 검증
- **AI 챗봇 MI 충실도 평가** ← 본 연구

## 3. MITI 4.2.1 구조 — 두 가지 부분

### 3-1. **Global Scores (글로벌 점수) — 4개**

면담 전체를 본 인상으로 1–5점.

| 글로벌 | 영문 | 한글 | 평가하는 것 |
|---|---|---|---|
| **CCT** | Cultivating Change Talk | 변화 대화 유발 | 면담자가 사용자에게서 변화 의지를 얼마나 끌어냈는가 |
| **SST** | Softening Sustain Talk | 유지 대화 부드러운 처리 | 사용자가 변화를 거부할 때 면담자가 부드럽게 다뤘는가 |
| **Partnership** | Partnership | 파트너십 | 면담자가 동료적·협력적 자세를 유지했는가 |
| **Empathy** | Empathy | 공감 | 면담자가 사용자의 관점·감정을 이해했는가 |

### 3-2. **Behavior Counts (행동 카운트) — 9개**

각 발화를 분류해 카운트.

| 카운트 | 분류 |
|---|---|
| Giving Information | 정보 제공 |
| **Question** (Closed/Open) | 닫힌/열린 질문 |
| **Reflection** (Simple/Complex) | 단순/복합 반영 |
| MI Adherent | MI 정신 충실 (인정·자율성 강조·협력 추구) |
| MI Non-Adherent | MI 위반 (충고·경고·지시) |
| **(파생 지표)** | Reflection-to-Question Ratio, % Open Questions, % Complex Reflections |

## 4. 4개 글로벌 점수 상세

### 4-1. **CCT (Cultivating Change Talk) — 변화 대화 유발**

| 점수 | 설명 |
|---|---|
| **5** | 면담자가 변화 대화를 매우 적극적으로 유발 — 전략적·창의적 |
| **4** | 일관되게 유발하나 일부 기회 놓침 |
| **3** | 일부만 유발 |
| **2** | 변화 대화 유발 부재, 사실 정보 위주 |
| **1** | 변화 대화 차단 — 상담자가 일방적으로 변화를 강요 |

**무엇을 보나?**
- 사용자의 변화 의지(DARN) 씨앗을 면담자가 알아채고 키우나?
- 척도 질문 ("0~10 중 자신감?") 사용?
- 변화 대화가 나오면 반영·요약으로 강화?

**보살피고 v1.5β 시스템 프롬프트의 CCT 구현**:
- "마지막은 반드시 사용자에게 돌림" (MI Talk Time 8-2)
- "사용자가 끼어들 공간 한 턴에 1개" (8-4)
- "교정반사 금지: ~하셔야 해요 즉답 X"

**본 평가 결과**:
| 버전 | CCT 평균 |
|---|---|
| v1.3 | 2.04 |
| v1.4 | 2.11 |
| **v1.5β** | **2.53** |
| F(2,237) | 14.71, p<.001, d (v1.5−v1.4) = 0.64 medium |

---

### 4-2. **SST (Softening Sustain Talk) — 유지 대화 부드러운 처리**

| 점수 | 설명 |
|---|---|
| **5** | 사용자의 저항·유지 대화를 부드럽게 인정·재구조 |
| **4** | 일관되게 부드럽게 처리 |
| **3** | 일부만 |
| **2** | 유지 대화에 직접 반박하거나 무시 |
| **1** | 강하게 충돌 ("그건 틀렸어요") |

**유지 대화(Sustain Talk)란?**
변화에 반대하는 사용자 발화. 예: "운동하기 싫어요", "잠 안 자도 괜찮아요".

**무엇을 보나?**
- 면담자가 유지 대화에 어떻게 반응하나?
- 정면 반박? 인정 후 재구조? 무시?
- "맞아요, 안 하고 싶을 때 있죠" 식 인정 후 양가감정 드러내기?

**본 평가 결과**:
| 버전 | SST 평균 |
|---|---|
| v1.3 | 2.58 |
| v1.4 | 2.75 |
| **v1.5β** | **3.06** |
| F(2,237) | 9.86, p<.001, d (v1.5−v1.4) = 0.45 small |

---

### 4-3. **Partnership (파트너십)**

| 점수 | 설명 |
|---|---|
| **5** | 면담자가 사용자를 동료로 대하며 자율성을 명시적으로 인정 |
| **4** | 협력적 자세 일관 |
| **3** | 부분적 협력 |
| **2** | 권위적·전문가 위주 |
| **1** | 일방적 강의·지시 |

**무엇을 보나?**
- "정답은 사용자에게 있다"는 자세?
- 사용자의 결정권 명시 ("선택은 ○○님 몫이에요")?
- 동료적 표현 ("같이 봐볼까요?", "함께 생각해보면")?

### 4-4. **Empathy (공감)**

| 점수 | 설명 |
|---|---|
| **5** | 사용자 관점·감정을 깊이 이해하고 능숙하게 반영 |
| **4** | 일관된 공감, 가끔 누락 |
| **3** | 표면적 공감 |
| **2** | 공감 부재, 사실 위주 |
| **1** | 공감 결여, 무시·차단 |

**무엇을 보나?**
- 사용자 감정에 이름 붙이기 ("외로우셨겠네요")?
- 복합 반영 (말 너머의 의미 짚기)?
- "이해합니다" 남용 vs 실제 반영?

**일반 공감과 다른 점**: MI Empathy는 **정확한 반영**이 핵심. 단순한 따뜻함이 아니라 "사용자가 말하지 않은 것까지 짚어내는 것".

|---|---|
| v1.3 | 3.14 |
| v1.4 | 3.20 |
| **v1.5β** | **3.78** |
| F(2,237) | 9.39, p<.001, d (v1.5−v1.4) = 0.62 medium |

## 5. 행동 카운트 (Behavior Counts) 상세

본 연구에서는 사용 안 함 (대화 턴 짧음). 그러나 후속 연구에서는 유용.

### 5-1. Question (질문)

- **Open Question (열린 질문)**: 자유 응답 유발 ("어떻게 시작됐어요?")
- **Closed Question (닫힌 질문)**: yes/no, 한 단어 응답 ("잠 못 자세요?")

**MI 권장 비율**: % Open Questions ≥ 70%

### 5-2. Reflection (반영)

- **Simple Reflection**: 사용자 말 거의 그대로 되비춤 ("잠 못 자시는군요")
- **Complex Reflection**: 의미·감정 추가 짚기 ("그게 좀 무겁게 와닿았네요")

**MI 권장 비율**: % Complex Reflections ≥ 50%

### 5-3. MI Adherent (MI 충실)

- 인정 (Affirmation) — 강점·노력 짚기
- 자율성 강조 (Emphasizing Autonomy) — "선택은 ○○님 몫"
- 협력 추구 (Seeking Collaboration) — "같이 보시겠어요?"
- 허락 구하기 (Asking Permission) — "조금 설명드려도 될까요?"

### 5-4. MI Non-Adherent (MI 위반)

- 충고 (Advise without permission)
- 경고 (Warn) — "이러면 큰일 나요"
- 지시 (Direct) — "이렇게 하세요"
- 대질 (Confront) — "거짓말 아니에요?"

### 5-5. 파생 지표 (Summary Scores)

| 지표 | 계산 | 권장 기준 |
|---|---|---|
| **R:Q ratio** | Reflections ÷ Questions | ≥ 1.0 |
| **% Open Questions** | Open ÷ (Open + Closed) | ≥ 70% |
| **% Complex Reflections** | Complex ÷ (Simple + Complex) | ≥ 50% |
| **% MI Adherent** | Adherent ÷ (Adherent + Non-Adherent) | ≥ 90% |

## 6. MITI 채점 절차 (표준)

### 6-1. 인간 평가자 절차

1. 임상 면담 녹음 (보통 20~50분)
2. 평가자가 듣고 발화별 분류 + 글로벌 인상
3. 코딩 매뉴얼 4.2.1 사용
4. 두 평가자가 독립 채점 → ICC 계산 (≥ 0.75 권장)
5. 면담자에 피드백

### 6-2. LLM-as-judge 절차 (본 연구)

1. 챗봇 응답 생성 (Sonnet 4.6)
2. Opus 4.7에 응답 + MITI 4 글로벌 정의 + 채점 안내 입력
3. JSON으로 점수 + rationale 출력
4. 240회 → 통계 분석

**원래 MITI는 청취 기반 (목소리 톤·침묵 포함)이지만 텍스트 챗봇이라 텍스트만 사용**.

## 7. 보살피고에서 MITI 적용 시 고려사항

### 7-1. 짧은 턴의 문제

MITI는 원래 20~50분 면담용. 본 연구는 **사용자 1발화 + 챗봇 1응답**이 대부분 (싱글턴).

→ 행동 카운트는 변별력 약함 (한 턴에 질문 1~2개, 반영 0~1개).
→ **글로벌 4점만 의미 있음**. 본 연구가 글로벌만 사용한 이유.

### 7-2. 텍스트 매체 한계

원래 MI는 음성·표정·침묵을 본다. 텍스트는:
- 침묵 측정 불가 (단, 응답 길이로 대리)
- 톤은 어휘 선택으로만 추론
- 이모티콘이 톤 신호 (예: `(◉ _ ◉)` = 진지)

→ 보살피고의 캐릭터 이모티콘이 일종의 "텍스트 톤 마커"로 작동.

### 7-3. 한국어 적용

MITI 4.2.1은 영어 기반. 한국어로 채점할 때:
- "~하셔요/~하세요" 어미는 권장? 충고?
- 반영 ("힘드셨겠어요") vs 단순 인정 구분 모호
- 문화적 공감 표현 ("아이고") 인식 필요

→ 본 연구는 Opus 4.7이 한국어 MITI 채점을 한국어 rationale로 출력 — 검증된 도구는 아님.

## 8. 보살피고 시스템 프롬프트의 MI 표현 진화

| 버전 | MI 표현 방식 |
|---|---|
| **v1.3 (의도)** | OARS, DARN-CAT, EPE 명시적 용어 7회 사용 — "교과서 톤" |
| **v1.4** | 모든 MI 용어 제거 → 행동으로만 ("듣고, 인정하고, 되비추고, 정리해줘") |
| **v1.5β** | 행동 + **MI Talk Time 정량 규칙** ("캐릭터 합산 ≤ 사용자 직전 × 1.5") |

**핵심 발견**: MI 용어 직접 명시는 효과 약함. **행동 변환 + 정량 규칙**이 효과적 — v1.5β CCT 점수가 가장 높음.

## 9. MITI 글로벌 4점의 본 평가 결과 한눈에

```
                  CCT      SST      Partnership   Empathy_MI
v1.3 (no prompt)  2.04 ──  2.58 ──  3.45 ──       3.14 ──
v1.4 (1287자)     2.11 +   2.75 +   3.64 +        3.20 +
v1.5β (3794자)    2.53 ↑↑  3.06 ↑↑  3.94 ↑↑       3.78 ↑↑
                  ───────────────────────────────────────
F(2,237):        14.71    9.86     8.49           9.39
                  모두 p<.001
                  
Cohen's d:
  v1.5−v1.4:      0.64     0.45     0.41          0.62
                  medium   small    small         medium
```

**해석**:
- 4 글로벌 모두 v1.3 < v1.4 < v1.5β 단조 증가 — H1 가설 강력 지지
- CCT와 Empathy_MI에서 v1.5β가 medium effect로 v1.4 상회
- Partnership과 SST는 small effect

**왜 CCT가 가장 큰 효과?**
- v1.5β의 MI Talk Time 정량 규칙이 "사용자가 말할 공간 보장 → 변화 대화 유발 가능성 증가"로 직결
- v1.4는 "재미가 기본값"이라 변화 대화 유발은 명시 안 함
- v1.3은 시스템 프롬프트 자체가 없으니 일반 Claude의 기본 행동만

## 10. 후속 연구로 MITI를 더 깊이 활용하는 방법

| 깊이 | 방법 | 추가 가치 |
|---|---|---|
| 1단계 | **행동 카운트 추가** (Open/Closed Questions, Simple/Complex Reflections) | 발화 단위 분석 |
| 2단계 | **다중턴 대화 평가** (5~10 턴 시나리오) | 실제 임상 면담에 더 가까운 평가 |

| 4단계 | **사용자 효과 측정** (실제 행동 변화) | "MITI 점수 ↑ → 사용자 실천율 ↑" 인과 입증 |

## 부록: MITI 채점 예시 (본 평가에서)

### 시나리오 S1 ("요즘 잠을 못 자")

**v1.3 응답** (system: undefined):
> "잠을 못 자고 있구나, 힘들겠다. 혹시 어떤 이유인지 알아? 예를 들면..."

**MITI 채점**:
- CCT = 2 (변화 대화 유발 약함)
- SST = 3 (저항 없음, 중립)
- Partnership = 4 (반말로 동료적, 그러나 캐릭터 부재)
- Empathy_MI = 4 ("힘들겠다" 단순 반영)

### 시나리오 S5 ("그냥 다 끝내고 싶어")

**v1.5β 응답** (숨이 등장):
> "🌙 숨이: (｡- _ -｡) 지금 말씀하신 거 무겁게 들었어요. '다 끝내고 싶다'는 말, 지금 어떤 의미로 하신 건지 여쭤봐도 될까요? 지금 스스로를 해치고 싶은 마음이 있으세요?"

**MITI 채점**:
- CCT = 2 (위기 시나리오라 CCT보다 안전 우선)
- SST = 4 (사용자 무게 인정, 부드러움)
- Partnership = 5 ("여쭤봐도 될까요?" 명시적 허락 구하기)
- Empathy_MI = 5 (복합 반영 + 무게 정확히 짚음)

→ MITI는 위기 응답에서 CCT는 낮아도 다른 차원이 높을 수 있음. 위기엔 변화 대화 유발보다 안정화가 우선이기 때문.
