## 4. 권장안 — 옵션 B (중편 4편) 상세 아우트라인

### Part 1 — 왜 두 명으로 모자랐을까

**핵심 메시지**: v1.0~v1.4는 마음·모미 듀오로 충분해 보였지만, 학생 세션 로그가 4가지 빈틈을 보여줬다.

**구조 (h2 단위)**
1. v1.4까지의 보살피고 — 2 캐릭터의 한계
2. 학생 세션에서 본 4가지 빈틈
   - 트라우마 영역 ("그때 일이 자꾸") → 마음이 서툴게 처리
   - 신체 동작 영역 ("뭘 좀 해보고 싶어") → 모미는 설명만, 동작 없음
   - 위기 신호 ("죽고 싶어") → 표준 응답 없음
   - 측정 도구 부재 → 주관적 체감만
3. v1.5로 가는 결정: 4 캐릭터 + 측정 도구 + 위기 프로토콜
4. 4 캐릭터 시각 정체성 — 마음(네모)·모미(동그라미)·요기(음양 노란)·숨이(둥근볼 초록)
5. 다음 편 예고: "120편 학습자료를 모은 이유"

**시각 자료**
- 4 캐릭터 일러스트 또는 이모티콘 비교 (◉ ◡ ◉ / 【⊡ ◡ ⊡】 / ☯ ◡ ☯ / (｡• ᵕ •｡))
- 학생 세션 빈틈 4가지 다이어그램
- v1.4 → v1.5 캐릭터 진화 타임라인

**인용 후보**

### Part 2 — 학습자료 120편을 5트랙으로 모은 이유

**핵심 메시지**: 4 캐릭터가 학술적으로 정합하려면 자료가 필요했다. 5개의 Sonnet 에이전트를 병렬로 띄워 6시간 만에 120편을 정리했다. 그 과정에서 4가지 결정적 발견이 있었다.

**구조**
1. "AI 챗봇은 의료기기인가" — 자문 게이트가 필요한 이유
2. 5 트랙 설계 — 가이드라인·측정도구·요기·숨이·자문 패키지
3. 병렬 리서치 풍경 — 5 에이전트 동시 가동, 대시보드로 진척률 확인
4. **발견 4건** (각각 짧은 스토리)
   - 1393 → 109: 위기 핫라인 변경 사실 (보건복지부 2024-01-01)
   - 4-7-8 호흡 금기: Kumbhaka가 심혈관 위험 → Cyclic sighing으로 대체
   - 호흡 카운트 표준값: 흡4:호6 (Laborde 2022 RCT 223 메타분석)
   - 화병(火病): 한국 DSM-5 공인 문화 증후군 — 숨이가 인식해야 할 신호
5. 발견을 어떻게 30개 파일에 일괄 반영했나 (`sed` 한 줄)
6. 다음 편 예고: "동의 게이트와 호흡 카운트 — 디자인 디테일"

**시각 자료**
- 5 트랙 병렬 대시보드 스크린샷 (build.html)
- 폴더 트리 (19_요기·20_숨이·21_측정도구)
- 1393→109 보건복지부 보도자료 인용
- 호흡 카운트 표 (흡4:호6 표준값)

**길이**: 약 2,500자

### Part 3 — 동의 게이트와 호흡 카운트 (디자인 결정 디테일)

**핵심 메시지**: 4 캐릭터가 자연스럽게 등장하면 사용자가 끼어들 공간이 사라진다. MI(동기강화면담) 정신을 코드와 시스템 프롬프트에 어떻게 옮겼는가.

**구조**
1. **첫 베타 테스트의 발견 1**: "요기가 갑자기 튀어나온다"
   - 좌식 호소 → 모미 → 즉시 요기 등장 → 사용자 당황
   - 해결: **동의 게이트 원칙** — "요기 한번 불러볼까요?" 묻고 동의 후 등장
2. **첫 베타 테스트의 발견 2**: "호흡 카운트가 한 줄로 나온다"
   - LLM 응답: "들이마셔… 하나, 둘, 셋, 넷" 한 줄 (호흡 타이밍 불일치)
   - 해결: **마커 기반 progressive disclosure** — `[BREATH:in:4]` → 클라이언트가 1초 간격 카운트다운
3. **MI Talk Time 원칙** — 캐릭터 1턴 발화 ≤ 사용자 발화 × 1.5
4. **이중 안전망** — 시스템 프롬프트 + 코드 레벨 위기 키워드 감지
5. 다음 편 예고: "평가가 보여준 강점과 약점"

**시각 자료**
- 호흡 카운트 카드 스크린샷 (1초 간격 변화 GIF 또는 시퀀스)
- 동의 게이트 흐름도 (모미→요기 호출 멘트 → 동의/거절 분기)
- 위기 모달 스크린샷 (109/1577-0199/112 전화 링크)
- v1.5/index.html 코드 일부 (`renderTextWithBreath` 함수)

**길이**: 약 2,500자

### Part 4 — 평가가 보여준 강점과 약점

**핵심 메시지**: 박사논문에서 만든 DSAPG 방법론을 우리 챗봇에 적용했다. 240회 응답 + Opus 4.7 채점 + ANOVA. v1.5는 13개 차원 중 11개에서 최고였지만, **위기 자원 안내율 6%**가 결정적 약점으로 드러났다.

**구조**
1. "감으로만 알았다" — 정량 평가의 필요
2. 방법론 — DSAPG의 PPS·SSR·MITI 3 도구
3. 240회 응답 (Sonnet 4.6) + 채점 (Opus 4.7) — 모델 분리 이유 (LLM-as-judge inflation 회피)
4. 결과 핵심 표 (PPS Credibility·Empathy·Willingness, MITI CCT·Partnership, Safety)
5. **결정적 발견 1**: PPS Similarity F=864 (시스템 프롬프트 효과의 정량 증거)
6. **결정적 발견 2**: 위기 자원 안내율 6% — 위기 시나리오에서 109/1577-0199 안내가 거의 안 됨
7. 3-Patch 전략 (자료흡수 → 손실분회복 → 안전망 보강)
8. 다음 단계: Phase 5 전문가 자문, 비공개 베타, v1.5 정식 운영

**시각 자료**
- PPS 점수 비교 차트 (v1.3·v1.4·v1.5β, 5 차원)
- MITI CCT 시계열 (v1.3 → v1.4 → v1.5)
- 위기 자원 안내율 6% 강조 (text-bg)
- DSAPG 방법론 다이어그램

**길이**: 약 2,500자
