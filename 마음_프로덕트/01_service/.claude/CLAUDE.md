# 01_service — 보살피고 서비스 개발 프로젝트

## 프로젝트 정의

실제 동작하는 **대화형 AI 에이전트(챗봇)** 의 개발·운영 자산.
브랜드명은 **보살피고(BOSALPIGO)**, 에이전트는 4종(모미·마음·숨이·요기).

## 범위 (이 폴더 안에서만 작업)

- 에이전트 캐릭터 정의 / 프롬프트 / 톤
- 사용자 여정 · 온보딩 · 대화 플로우
- MI(동기강화면담) 가이드
- 예외 대화 시나리오 · 평가 계획
- 근거 레이블링 규칙
- 수업·세션 로그(`logs/`)
- 버전 비교 리포트(`reports/`)

> **주의**: 지식 아카이브(기존 `research/`)는 `../../자료/04_캐릭터_학습자료/` 로 이동됨 (2026-04-24).
> 자동 수집된 신규 지식은 `../../자료/06_캐릭터별_배포/<캐릭터>/` 참조.

## 작업 규칙

1. **지식 아카이브 참조**: `../../자료/04_캐릭터_학습자료/` (원본), `../../자료/06_캐릭터별_배포/` (자동수집 후 캐릭터별 분배본) — **읽기 전용**
2. 버전 업데이트 시 `reports/` 에 비교 리포트 남기기
3. 로그(`logs/`)는 익명화된 상태로만 저장
4. 브랜드 자산(컬러·타이포·네이밍)이 필요하면 `../02_brand_campaign/` 을 **읽기 전용으로 참조** — 수정은 거기서만
5. 블로그 콘텐츠로 쓸 작업 기록은 `../03_blog_process/` 에 요약 공유 가능, 단 그쪽에서 이 폴더를 수정할 수 없음

## 다른 프로젝트와의 관계

- `../02_brand_campaign/` — 브랜드 시스템 정의. 본 프로젝트가 따르는 브랜드 규칙의 출처.
- `../03_blog_process/` — 블로그/제작기록. **이 폴더 내용을 읽기만** 한다.
