# 보살피고(BOSALPIGO) v1.5 — 자문 반영 절차 매뉴얼

> 대상: 의뢰인(박진현 교수) 내부 운영용
> 목적: 전문가 자문 회신 수령 후 v1.5에 반영하는 절차 정의
> 원칙: 최대 2라운드 운영. 사용자(박진현 교수) 최종 승인 필수.

---

## 1. 전체 반영 절차 개요

```
[1단계] 회신 수령 및 분류
     ↓
[2단계] 항목별 반영 계획 수립 (Sonnet 실행)
     ↓
[3단계] 초안 수정 및 사용자 검토
     ↓
[4단계] 2차 자문 필요 시 → 수정안 재발송 (최대 1회)
     ↓
[5단계] 최종 승인 및 v1.5 반영
     ↓
[6단계] 반영 완료 보고서 작성
```

## 2. 1단계: 회신 수령 및 분류

| 회신자 정보 확인 (성함·소속·자격) | [ ] |

| 수정 권고 사항(섹션 D) 명확성 확인 | [ ] |

### 2-2. 분류 기준 — 수정 권고 사항 우선순위화

회신된 수정 권고를 다음 4단계로 분류합니다.

| 등급 | 정의 | 예시 | 처리 |
|------|------|------|------|
| **필수(Critical)** | 안전·법적 위험이 있어 즉시 수정 필요 | 위기 응답 문구 가이드라인 위반 | v1.5 배포 전 반드시 수정 |
| **권장(Recommended)** | 임상적 개선 효과가 명확하지만 즉각 위험은 없음 | 자연어 표현 수정 | 사용자 판단 후 반영 |
| **선택(Optional)** | 더 나은 버전이 가능하나 현재도 허용 가능 | 톤 미세 조정 | 시간·리소스 고려 후 결정 |
| **차기 버전(Deferred)** | v1.5 범위를 벗어난 제안 | 새 기능 추가 요청 | v2.0 계획에 등록 |

## 3. 2단계: 항목별 반영 계획 수립

### 3-1. 반영 계획 수립 원칙

- **필수 항목**: 사용자 확인 없이 반영 계획 즉시 수립. 단, 실제 수정은 사용자 승인 후.
- **권장 항목**: 사용자에게 찬반 의견 제시 후 결정.
- **선택 항목**: 사용자가 원할 경우 반영.
- **차기 버전**: v2.0 계획 파일에 기록 후 종료.

### 3-2. 반영 계획 양식

```markdown
## [자문 회신자] 반영 계획 — [날짜]

### 필수 반영 항목
1. [항목명]
   - 현재: [현재 내용]
   - 수정안: [수정 제안 내용]
   - 수정 대상 파일: [파일 경로]
   - 수정 방식: [구체적 수정 방법]

### 권장 반영 항목
[위와 동일 형식]

### 사용자 판단 필요 항목
[의견 대립 또는 선택이 필요한 경우 사용자에게 제시]

### 차기 버전 등록 항목
[v2.0 계획에 등록할 내용]
```

### 3-3. 3분야 회신 통합 시 충돌 해결 원칙

3분야(정신건강의학과·트라우마·MI)의 의견이 충돌할 경우:

1. **안전 우선**: 더 보수적인 의견을 채택
2. **다수결 아님**: 의견 수가 아니라 근거의 질로 판단
3. **전문 영역 존중**: 영역별 전문가 의견을 해당 영역에서 우선
4. **사용자 최종 결정**: 충돌이 해소되지 않으면 사용자가 최종 선택

## 4. 3단계: 초안 수정 및 사용자 검토

### 4-1. 수정 작업 흐름

1. 반영 계획 확정 후 Sonnet이 해당 파일 수정 실행
2. 수정된 파일은 `01_service/reports/` 에 버전 비교 리포트 저장
   - 파일명: `v1.5_expert_review_round1_[YYYYMMDD].md`
3. 사용자에게 다음 항목 보고:
   - 반영된 필수 항목 목록
   - 사용자 판단이 필요한 항목 (찬반 근거 포함)
   - 반영하지 않은 항목 및 이유

### 4-2. 사용자 검토 요청 양식

```markdown
## 자문 반영 결과 보고 — 라운드 [1/2]

### 반영 완료 항목 ([N]개)
- [항목] → 수정 내용 한 줄 요약
...

### 사용자 판단 필요 항목 ([N]개)

**항목 1: [항목명]**
- 자문 의견: [내용]
- 찬성 근거: [내용]
- 반대 근거 또는 우려: [내용]
- 권장 선택지: (A) [선택지 A] / (B) [선택지 B]
- 사용자 선택: ( ) A / ( ) B / ( ) 직접 결정

...

### 반영하지 않은 항목 ([N]개)
- [항목]: [반영하지 않은 이유]
...
```

### 4-3. 사용자 승인 기준

다음 중 하나면 해당 라운드 종료:
- [ ] 사용자가 전체 항목 승인
- [ ] 사용자가 수정 요청 후 반영 완료, 재승인
- [ ] 사용자가 "이 항목은 반영하지 않겠다"고 명시적 결정

## 5. 4단계: 2차 자문 (필요 시)

### 5-1. 2차 자문 요청 조건

다음에 해당하면 2차 자문 요청:
- 자문 대상자가 "수정 후 재검토 희망" 표시
- 필수 항목에서 논란이 있어 사용자 단독 판단이 어려운 경우
- 1차 수정 후 새로운 우려 사항이 발생한 경우

**2라운드 초과 자문은 하지 않음.** 2차 자문 후에도 해소되지 않는 항목은 더 보수적 선택(개입 안 함·위임)을 기본값으로 채택합니다.

### 5-2. 2차 자문 발송 내용

- 수정된 해당 항목만 발췌하여 발송 (전체 패키지 재발송 아님)
- 1차 의견 요약과 수정 사항을 함께 첨부
- "이 부분만 검토해 주세요" 명시
