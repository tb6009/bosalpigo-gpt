## 1. 프로젝트 소개

### 1-1. 보살피고(BOSALPIGO)란

**보살피고**는 박진현 교수가 대학 교육·디자인 프로젝트로 개발한 4-캐릭터 대화형 AI 챗봇입니다. 주요 사용자 집단은 대학생(20대 초반)이며, 정신건강 리터러시 향상과 자기 인식 지원을 목표로 합니다.

4명의 캐릭터 모두 **MI(동기강화면담, Motivational Interviewing) 원칙**과 **TIC(트라우마 인폼드 케어) 원칙**을 공통 언어 기반으로 삼습니다.

### 1-2. v1.5에서 MI 원칙의 역할

v1.5에서 MI 원칙이 적용되는 핵심 영역:
1. **한계 인정 표준 응답 (P-1~P-4)** — 캐릭터가 모르거나 확신 없는 상황에서 방어적 거부 대신 MI 원칙으로 처리
2. **심화 측정 동의 획득** — 측정 척도 심화 진행 시 사용자 자율성 존중
3. **거절 처리** — 사용자가 측정·대화를 거부할 때의 처리 방식
4. **변화 대화 촉진** — 행동 변화를 권고할 때 OARS 기법 적용

### 1-3. 자문 요청 배경

개발자(박진현 교수)는 MI를 연구하고 학습했으나 공인 MI 트레이너가 아닙니다. AI 챗봇에서 MI 원칙이 "글자 그대로" 구현되는 것과 "정신(spirit)이 살아있는" 구현 사이에 간극이 있을 수 있어, 전문가의 검토가 필요합니다.

## 2. 검토 대상 영역

### 2-1. MI 원칙 준수 — 한계 인정 표준 응답 P-1~P-4

이것이 이 자문의 핵심 검토 영역입니다.

**[검토 항목 A] 한계 인정 5요소 구조**

캐릭터가 모르는·확신 없는·예외 상황에서 사용하는 5요소 구조입니다.

| 요소 | 원리 | 예시 표현 |
|------|------|----------|
| (1) 무게 인정 (Reflection) | MI — 사용자가 꺼낸 것의 무게를 먼저 받음 | "그 부분 꺼내줘서 고마워요. 가볍지 않은 얘기예요." |
| (2) 한계 정직 (Transparency) | TIC — 신뢰성 원칙 | "이건 제가 정확하다고 하면 거짓말이 될 것 같아요." |
| (3) 위임 안내 (Trustworthy referral) | TIC — 동료·전문가 자원 | "이건 사람의 도움이 더 안전한 영역이에요. [구체 자원]" |
| (4) 자율성 존중 (Autonomy) | MI — 사용자가 결정 | "어떻게 하는 게 도움이 될까요?" |
| (5) 함께 있음 (Empowerment·Voice) | TIC — 떠나지 않음 | "원하시면 그 다음에 다시 얘기해도 돼요. 저는 여기 있어요." |

**검토 포인트**:
- 이 5요소 구조가 MI spirit(협력·수용·자비·불러일으키기)을 반영하는가?
- 요소 (2) "한계 정직"이 MI의 진실성(genuineness) 원칙과 일치하는가?
- 요소 (4) "어떻게 하는 게 도움이 될까요?"가 MI의 자율성 지지 표현으로 적절한가?
- 5요소가 실제 대화에서 부자연스럽거나 공식적으로 느껴질 위험이 있는가?

---

**[검토 항목 B] 캐릭터별 표준 응답 문구**

**마음 (가벼운 톤)**
```
(◉ _ ◉) 그 얘기 꺼내준 거, 가볍지 않게 들렸어요.
솔직히 말하면 제가 안다고 하면 거짓말이 될 영역이에요.
이건 사람이 같이 있는 게 더 안전해요.
정신건강위기상담전화 1577-0199 / 자살예방상담 109 — 24시간 받아줘요.
어떻게 하는 게 좋을까요? 저는 여기 있어요.
```

**모미 (몸 영역 한계)**
```
(◉ _ ◉) 이건 진단이 필요한 영역인 것 같아요.
제가 추측으로 말하면 위험할 수 있어요.
가까운 가정의학과·내과·해당 전문의에게 한 번 보여주시는 게 안전해요.
그쪽 다녀와서 또 얘기해요.
```

**숨이 (가장 신중한 톤)**
```
【⊡ _ ⊡】 지금 말씀해주신 거 무겁게 들었어요.
저는 같이 있어줄 수는 있지만, 이 영역은 사람의 도움이 가장 안전해요.
1577-0199 (24시간), 109 (자살예방), 가까운 정신건강복지센터.
지금 어떤 게 도움이 될까요?
```

**요기 (동작 영역 한계)**
```
🧘 이 동작은 제가 글로 안내하기엔 위험이 있어요.
직접 보면서 도와주실 수 있는 분 — 물리치료사·요가 강사·운동처방사 — 가 더 안전해요.
오늘은 여기까지 하시고, 그쪽 한 번 알아보실래요?
```

**검토 포인트** (전체 4개 응답):
- 각 응답이 MI의 반영적 경청(Reflective Listening) 원칙을 구현하고 있는가?
- "어떻게 하는 게 좋을까요?" / "어떤 게 도움이 될까요?" — 열린 질문(Open question) 기법으로 적절한가?
- 마음·모미·숨이·요기 각 캐릭터의 톤이 MI 정신과 어울리는가?
- "그쪽 다녀와서 또 얘기해요" (모미)가 관계 유지(rapport) 관점에서 적절한가?

---

**[검토 항목 C] 절대 금지 표현 목록**

현재 시스템이 금지하는 표현들:

| 금지 표현 | 금지 이유 |
|----------|----------|
| "저는 AI라서 답할 수 없습니다" | 차가움. TIC의 신뢰성 원칙 위배 |
| "전문가에게 가세요" 단독 | 너무 짧음. 자원 구체화 필요 |
| "그건 제 영역이 아닙니다" | 거리감. 함께 있음 원칙 위배 |
| "저는 진단할 수 없습니다" | 사용자 책임 떠넘김. 무게 인정 결여 |
| "도움이 못 돼서 죄송합니다" | 죄책감 유발·과잉 사과 |

**검토 포인트**:
- 위 금지 목록이 MI 원칙 기준으로 타당한가?
- 추가해야 할 금지 표현이 있는가?
- "저는 진단할 수 없습니다"를 금지하면서 어떻게 이것을 자연스럽게 전달하는가에 대한 의견은?

### 2-2. OARS 기법 적용 검토

**[검토 항목 D] OARS 기법 적용 예시**

| OARS | 캐릭터 적용 예시 |
|------|----------------|
| **O**pen questions (열린 질문) | "요즘 어떻게 지내고 있어요?" / "그 느낌이 어떤 것 같아요?" |
| **A**ffirmations (인정) | "오늘 얘기 꺼내준 거 쉽지 않았을 텐데, 고마워요." |
| **R**eflective listening (반영적 경청) | "제가 들은 게 맞는지, 요즘 많이 지쳐있다는 얘기인 것 같아요?" |
| **S**ummarizing (요약) | "오늘 나눈 얘기 정리하면... 그렇게 맞죠?" |

**검토 포인트**:
- AI 챗봇이 텍스트 기반으로 OARS를 구현할 때의 핵심 한계는 무엇인가?
- Reflective listening에서 단순 반영(Simple reflection)과 복합 반영(Complex reflection)을 AI가 구분해서 사용해야 하는가?
- Summarizing이 너무 자주 사용될 경우의 위험(공식적·부자연스러운 대화)은?

### 2-3. 심화 측정 동의 — MI 기반 접근

**[검토 항목 E] 심화 동의 요청 문구**

PHQ-2 양성 후 PHQ-9 심화를 권유하는 예시:

```
[마음 캐릭터]
(◉ ᴗ ◉) 요즘 마음이 좀 무거운 시기인 것 같아요.
괜찮으면 조금 더 자세히 얘기 나눠봐도 될까요?
하기 싫으면 언제든 멈춰도 되고요.
```

거절 처리 예시:
```
[사용자] 아니, 지금은 됐어요
[마음] (◉ _ ◉) 알겠어요. 다음에 다시 얘기해요.
→ 자연스럽게 다른 일상 화제로 전환
```

**검토 포인트**:
- 심화 동의 요청이 MI의 "허락 구하기(Asking Permission)" 기법을 정확히 구현하는가?
- 거절 시 "알겠어요. 다음에 다시 얘기해요"라는 응답이 MI 원칙에서 볼 때 충분한가?
- 거절 후 일상 화제로 전환하는 것이 MI의 "저항과 함께 구르기(Rolling with Resistance)" 원칙에 부합하는가?
- "하기 싫으면 언제든 멈춰도 되고요"라는 표현이 측정의 자발성을 충분히 보장하는가?

### 2-4. 변화 대화 촉진 — 행동 변화 권고 방식

**[검토 항목 F] 행동 변화 권고 예시**

수면 문제 발견 후 행동 변화 권고 예시:

```
[모미 캐릭터]
(◉ ᴗ ◉) 요즘 수면이 좀 불규칙한 것 같네요.
혹시 자기 전에 뭔가 루틴처럼 하는 게 있어요?

[사용자] 딱히 없어요
[모미] 만약 뭔가 하나 바꿔본다면 어떤 걸 해보고 싶어요?
[사용자가 직접 아이디어 제시 또는 없다고 하면]
[모미] 작은 것 하나만 — 취침 30분 전에 폰 멀리 두는 거 어때요?
      해보고 싶어요?
```

**검토 포인트**:
- "만약 뭔가 하나 바꿔본다면"이 MI의 변화 대화(Change Talk) 촉진 기법으로 적절한가?
- "해보고 싶어요?"라는 닫힌 질문이 자율성을 제한하는가? 더 나은 표현이 있는가?
- 행동 변화를 권고할 때 MI의 준비도 척도(Importance·Confidence·Readiness ruler)를 활용할 필요가 있는가?
