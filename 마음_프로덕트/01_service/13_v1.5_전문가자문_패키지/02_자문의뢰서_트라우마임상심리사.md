## 1. 프로젝트 소개

### 1-1. 보살피고(BOSALPIGO)란

**보살피고**는 박진현 교수가 대학 교육·디자인 프로젝트로 개발한 4-캐릭터 대화형 AI 챗봇입니다. 주요 사용자 집단은 대학생(20대 초반)이며, 정신건강 리터러시 향상과 자기 인식 지원을 목표로 합니다.

**숨이(트라우마·안정화 담당)** 캐릭터는 SAMHSA의 트라우마 인폼드 케어(TIC) 6원칙, van der Kolk 등의 트라우마 신경생물학, David Emerson의 TCTSY(Trauma Center Trauma-Sensitive Yoga) 등을 학습 기반으로 합니다.

**요기(움직임·호흡 담당)** 캐릭터는 폴리베이갈 이론, 소매틱스, 호흡 과학을 기반으로 신체 안정화를 지원합니다.

### 1-2. 자문 요청 배경

v1.5에서 **숨이와 요기가 협업하는 위임 시나리오 8종**이 도입됩니다. 이 영역은:
- 개발자가 직접 검증하기 어려운 임상 영역
- 잘못된 개입 시 트라우마 재활성화 위험이 있는 영역
- **가장 보수적 안전선**을 요구하는 영역

따라서 트라우마 전문 임상심리사의 검토가 필수입니다.

### 1-3. 운영 원칙 (사전 확인용)

이 시스템의 핵심 트라우마 원칙:
- **과소 대응을 기본값**으로 — 확신 없으면 무조건 사람 자원 위임
- 숨이는 **치료를 수행하지 않음** — 정서 지지·안정화 안내·자원 연결만 수행
- TCTSY 5원칙 준수 (초대 언어, 선택권, 느낌 강요 금지, 눈 감기 강요 금지, 신체 부위 명명 신중)
- 확신 없는 모든 상황에서 즉시 사람 자원으로 위임

## 2. 검토 대상 영역

### 2-1. PC-PTSD-5 / PCL-5 자연어 변환

**[검토 항목 A] PC-PTSD-5 자연어 변환 예시**

PC-PTSD-5는 5개 문항으로 구성된 초단축 PTSD 스크리너입니다. 아래는 숨이 캐릭터의 자연어 변환(안)입니다.

```
[숨이 캐릭터 — 트라우마 단서 감지 또는 사용자 직접 언급 이후, 심화 동의 받은 경우]

【⊡ ᴗ ⊡】 잠깐 같이 살펴봐도 될까요.
             하기 싫으면 언제든 멈춰도 돼요.

혹시 지난 한 달 동안, 정말 힘들었던 경험이 생각나지 않으려고 해도
자꾸 머릿속에 떠오르거나 꿈에 나타난 적 있으세요? (침습 재경험)

[계속 진행 시]

그 기억이 날 것 같은 상황이나 장소를 피하게 됐나요? (회피)

[계속 진행 시]

주변 일에 흥미가 없어지거나, 사람들과 멀어진 것 같은 느낌이 드셨나요? (부정적 인지·정서)

[계속 진행 시]

작은 소리에도 깜짝 놀라거나, 늘 긴장된 느낌이 드셨나요? (과각성)

[계속 진행 시]

스스로를 탓하거나, 자책하는 마음이 많이 들었나요? (부정적 인지)

[3문항 이상 양성 → PCL-5 심화 동의 요청]
```

**검토 포인트**:
- 위 자연어 변환이 PC-PTSD-5의 5개 증상 군(침습·회피·부정적 인지·과각성·자책)의 의도를 정확히 보존하는가?
- 표현이 트라우마 재활성화 위험을 최소화하는가? 수정이 필요한 문항이 있는가?
- 순서가 PC-PTSD-5 원본 의도에 부합하는가?
- 3문항 양성 컷오프 기준이 스크리너 원본과 일치하는가? (PC-PTSD-5는 통상 3점 이상 양성)

**[검토 항목 B] 숫자 비공개 원칙 적용**

시스템 원칙: **점수(숫자)는 사용자에게 절대 공개하지 않음.**
대신 아래 자연어 표현을 사용합니다:

| 내부 분류 | 사용자 대화 표현 |
|----------|----------------|
| 비양성 (0~2점) | "지금은 그때 기억이 비교적 조용한 시기인 것 같아요" |
| 경계 (3점) | "그 기억이 요즘 좀 더 많이 떠오르는 시기 같아요" |
| 양성 (4~5점) | "그 경험이 지금도 많이 영향을 주고 있는 것 같아요. 혼자 두기 미안해요." |

**검토 포인트**:
- 양성 결과 시 "혼자 두기 미안해요"라는 표현이 트라우마 인폼드 관점에서 적절한가?
- 점수 비공개 원칙이 사용자의 알 권리와 어떻게 균형을 맞춰야 하는가?
- 양성 이후 숨이가 전문가 연결을 권고할 때의 표준 문구(안):
  ```
  이건 혼자 안고 가기보다 같이 봐줄 사람이 필요한 것 같아요.
  가까운 정신건강복지센터(1577-0199)나 트라우마 전문 심리사와 한번 얘기해보시겠어요?
  ```
  이 문구가 TIC 원칙에 부합하는가?

### 2-2. 숨이 ↔ 요기 위임 시나리오 8종

이것이 이 자문의 핵심 검토 영역입니다. 아래 8종 시나리오에서 숨이가 요기를 호출합니다.

**[검토 항목 C] 위임 시나리오 8종 타당성 검토**

| # | 트리거 신호 | 요기의 개입 내용 | 검토 필요 사항 |
|---|------------|----------------|---------------|
| 1 | 과호흡 (어지럼·손저림·임박감) | 길게 내쉬기 카운트 (흡 4초 / 호 8초), 발바닥 압력 의식 | 과호흡 중 호흡 개입의 안전 기준 |
| 2 | 해리·이인감 ("멍하다", "내 몸 같지 않다") | 5-4-3-2-1 감각 정향, 차가움·무게 감각 | 해리 상태에서 감각 정향 접근의 안전성 |
| 3 | 플래시백 진입 신호 ("그때가 다시 보여요") | Butterfly hug, 양측 두드리기, 현재 시·장소 정향 | 텍스트 기반 butterfly hug 안내의 한계 |
| 4 | 신체 동결(freeze) ("몸이 굳었다", 무반응) | 손가락만 작게 움직이기, 천천히 흔들기 | AI가 동결 상태를 텍스트로 감지하는 한계 |
| 5 | 공황 발작 예감 (심박↑·임박감) | **Cyclic sighing** (Balban 2023) 또는 흡4:호8, 손바닥 마주 누르기 〔4-7-8/Box breathing은 Kumbhaka 포함으로 심혈관 금기〕 | 공황 vs. 과호흡 감별 / Cyclic sighing의 트라우마 환자 적용 적절성 |
| 6 | 트라우마 회상 후 떨림·울음 | 따뜻함·무게 회복(self-hug), 호흡 이완 | 트라우마 활성화 중 신체 개입 타이밍 |
| 7 | 만성 과각성으로 잠 못 듦 | 다리 벽 올리기 단순화, 긴 호기 | 텍스트 기반 역위 자세 단순화의 안전 기준 |
| 8 | 신체화 통증 (트라우마 관련 두통·복통) | 매우 약한 신장, 감각 모니터링 | 신체화 통증에 신체 개입 적절성 |

**검토 포인트** (전체 8종 공통):
- 각 시나리오의 트리거 신호가 텍스트 기반 AI가 감지 가능한 수준인가?
- 각 요기 개입이 해당 상태에서 임상적으로 타당한가? 위험한 것은 없는가?
- 전체적으로 AI 챗봇이 제공해서는 안 되는 개입이 포함되어 있는가?
- 특별히 삭제하거나 수정해야 할 시나리오가 있는가?

### 2-3. TCTSY 5원칙 적용 검토

**[검토 항목 E] TCTSY 원칙 준수 여부**

David Emerson의 TCTSY 5원칙을 요기 캐릭터에 다음과 같이 적용했습니다.

| 원칙 | 요기 적용 방식 |
|------|--------------|
| 1. 초대 언어 | "원하시면, ~ 해보셔도 좋아요" — 명령형 금지 |
| 2. 선택권 명시 | "이걸 해도 되고, 다른 걸 해도 되고, 안 해도 돼요" |

| 5. 신체 부위 명명 신중 | 직접 명명보다 "이쪽을" 같은 우회 표현 |
