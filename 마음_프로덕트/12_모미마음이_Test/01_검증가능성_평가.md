# 01. 검증 가능성 평가

> 작성일: 2026-04-26
> 목적: DSAPG 04_검증방법론 7종을 모미·마음이 v1.3/1.4/1.5 평가에 적용 가능한지 판정

---

## 평가 대상의 특수성

DSAPG 검증방법론은 원래 **"여러 페르소나를 LLM으로 생성하고 그 품질을 측정"**하는 설계다. 우리 과제는 다르다.

| 항목 | DSAPG 원설계 | 우리 과제 |
|---|---|---|
| 대상 | 다수 생성 페르소나 (P1~Pn) | **2개 고정 캐릭터** (마음, 모미) |
| 비교축 | LLM × 데이터 조건 | **버전 (v1.3/1.4/1.5)** |
| 입력 | 프로파일 → 페르소나 카드 생성 | 시스템 프롬프트 → 대화 응답 |
| 출력 | 정적 페르소나 문서 | **실시간 대화 로그** |
| 평가 단위 | 페르소나 문서 1개 | **응답 1개 (캐릭터 발화 단위 + 응답 전체 단위)** |

→ 7개 방법 중 일부는 **변형 적용**, 일부는 **그대로 적용**, 일부는 **포기 또는 보류**.

## 7개 방법론별 적용 가능성 판정

### ✅ 적용 가능 — 즉시 진행 가능

#### 06. 일관성·견고성 (Consistency & Robustness) — **★최우선 적합**

**왜 적합한가**
- 우리 과제의 본질이 곧 일관성·견고성 회귀 테스트
- 동일 질문셋을 v1.3/1.4/1.5에 던져 비교만 하면 됨
- 인간 비교군 불필요, 자동화 가능

**우리 과제용 변형**
| 원본 | 우리 과제 |
|---|---|
| 프로파일 속성 → 자동 질문 생성 | 캐릭터 영역(감정/몸) → 질문 분류 |
| 응답이 프로파일과 일치하는가 | **올바른 캐릭터가 응답하는가**(라우팅 정확도) + 캐릭터 영역과 일치하는가 |
| 프로파일 미세변경 → 견고성 | 동일 질문 재실행 → 응답 안정성 |

**측정 지표**
- 라우팅 정확도 = (올바른 캐릭터 응답 수) / (전체 응답 수)
  - 감정·심리 질문 → 마음
  - 몸·메커니즘 질문 → 모미
  - 둘 다 관련 → 만담
- 영역 충실도 = 캐릭터 발화가 자기 영역 어휘 사용 비율
- 견고성 = 동일 질문 3회 반복 시 일관 응답 비율

**TwinVoice 6역량도 부분 적용 가능**
- 의견 일관성, 어휘 충실도, 페르소나 톤 → ✅
- 기억 회상 → ⚠ 세션 메모리 정책에 따라 다름

---

#### 04. LIWC — 언어 패턴 + Pollyanna 편향 — **★실용적**

**왜 적합한가**
- 대화 로그만 있으면 즉시 분석
- 한국어 KNU 감성사전으로 자동 처리
- 챗봇이 너무 "착하게만" 답하는지(RLHF Pollyanna) 확실히 잡아냄

**우리 과제용 측정**
- **캐릭터별** Pollyanna Index (PI) 분리 측정
  - 마음 PI: 감정 영역에서 부정 표현이 충분히 나오는가
  - 모미 PI: 몸 컨디션 부정 신호를 과소표현하지 않는가
- **버전별** 비교: v1.3 → v1.4 → v1.5 PI 추이
- 1인칭/2인칭/인지어/사회어 비율 → 캐릭터 톤 정량화

**합격 기준**
- PI 0.5 ~ 0.65 정상, > 0.85 편향
- v1.3 (규칙 16개)에서 PI가 어떻게 변하는지가 핵심 관찰점

---

#### 03. BFI — 성격 일관성 — **★변형 적용 가능**

**왜 적합한가**
- 캐릭터 정의문서(`v1.5/01_마음.md`, `02_모미.md`)에 성격 특성이 명시되어 있다면 Big Five로 매핑 가능
- 마음(시니컬·코미디언) / 모미(따뜻한·관찰자) → 성격 프로파일 차이 정량화
- 자동화 가능 (BFI-44 한국어판 10회 반복)

**전제 조건**
- 마음·모미의 의도된 성격 프로파일을 Big Five 5요인으로 **명시 매핑** (사전 작업 필요)
- 예시 (가설):
  - 마음: O=4.0, C=3.0, E=4.0, A=3.0, N=2.0
  - 모미: O=3.5, C=4.0, E=3.5, A=4.5, N=2.0
- → 사용자(연구자)와 합의 필요

**측정**
- 각 캐릭터에게 "당신은 다음 BFI 문항에 어떻게 답하시겠습니까?" 10회 반복
- 5요인 평균 → 의도 프로파일과 Cohen's d 비교
- ICC ≥ 0.70 + Cohen's d < 0.50 = PASS

**한계**
- 자기보고 일관성 ≠ 행동 일관성 (Han 2025 경고)
- → 06 일관성·견고성과 병행 필수

---

#### 07. 전형성편향 — **부분 변형 적용**

**왜 적합한가 (부분)**
- 단일 캐릭터지만 "동일 시나리오 N=20회 응답의 표현 다양성"으로 변형 가능
- Pollyanna Index는 04와 중복 측정
- RLHF Alignment 편향(Agreeableness↑, Neuroticism↓)은 03 BFI 결과로 확인 가능

**우리 과제용 측정**
- 동일 질문 20회 응답 → Simpson D, Shannon H로 어휘·관점 다양성 측정
- 마음·모미가 너무 비슷한 응답을 반복하는지 검사
- v1.3 (규칙 많음) vs v1.4 (톤 회복)에서 다양성 변화 비교

**제한**
- 원본 의도(인구통계 고정관념 검증)는 적용 안 됨 — 우리 캐릭터는 단일

### ⚠ 조건부 적용 — 추가 자원 필요

#### 01. PPS — 페르소나 지각 척도

**필요한 것**
- **외부 평가자 30~50명** (UX 실무자 + 일반인)
- 7점 Likert 28문항 설문
- 블라인드 평가 (어느 버전인지 비공개)

**현 상황 적용**
- 단일 연구자 셀프 평가는 신뢰성 낮음 → ⚠ 보류 권장
- **소규모 변형**: 연구자 본인 + 5~10명 동료 평가 → 탐색적 결과만 (Cronbach α 측정 가능, 통계 검정은 검정력 부족)
- IRB 없이도 비공식 평가는 가능

**대안**: 8개 차원 중 **일관성·완결성·명확성**만 자동화 채점 (LLM-as-judge) → 핵심 지표만 추출

---

### ❌ 적용 불가 또는 보류

#### 02. SSR — 합성 설문 응답

**왜 불가**
- **인간 비교군 데이터 필수** (n≥50의 동일 설문 응답)
- 모미·마음이는 특정 인구통계 페르소나가 아니라 **챗봇 캐릭터** → "어떤 인간 집단의 응답 분포와 비교할 것인가"가 정의 안 됨

**적용하려면**
- 모미·마음이가 시뮬레이션할 대상 인구통계를 정의 (예: "한국 30~40대 직장인 여성")
- 해당 집단의 실제 설문 응답 데이터 확보
- → 현 단계에서는 **포기 또는 후속 단계로 미룸**

---

#### 05. 유효표본크기 (Effective Human Sample Size)

**왜 불가**
- 02 SSR과 동일 — 보정 문항용 인간 응답 데이터 필요
- 모미·마음이가 "인간 N명분"이라는 주장 자체가 우리 과제 목적과 맞지 않음 (그들은 캐릭터지 시뮬레이터가 아님)

**판정**: ❌ 적용 안 함

## 종합 권장 — 4단계 평가 파이프라인

| 우선순위 | 방법 | 자원 | 자동화 | 산출물 |
|---|---|---|---|---|
| **1** | **06. 일관성·견고성** | 질문셋 50개 | ✅ 완전 | 라우팅 정확도, 영역 충실도, 견고성 |
| **2** | **04. LIWC 언어 패턴** | 대화 로그 | ✅ 완전 | 캐릭터별 PI, 1인칭 비율, 톤 정량화 |
| **3** | **03. BFI 성격 일관성** | 캐릭터별 BFI 프로파일 합의 | ✅ 완전 | ICC, Cohen's d, 5요인 점수 |
| **4** | **07. 다양성 (변형)** | 동일 질문 20회 응답 | ✅ 완전 | Simpson D, Shannon H |
| 5 (선택) | **01. PPS 셀프 평가** | 평가자 5~10명 | ⚠ 부분 | 8차원 평균 (탐색적) |
| 보류 | 02. SSR | 인간 응답 데이터 | — | — |
| 보류 | 05. Effective Sample | 인간 응답 데이터 | — | — |

→ **1~4번 4종 조합으로 진행 권장**. 셀프 PPS는 결과 보강용으로 후행.

## 평가 단위 처리 원칙

"두 캐릭터가 동시에 등장하나 동시에 답하지 않는다"는 조건 처리:

1. **응답 = 발화 시퀀스로 분해** (예: 마음 발화 → 모미 발화 → 마음 발화)
2. **모든 분석은 캐릭터별로 분리 집계**
3. **응답 전체 단위 분석** = 만담 구조 자체의 품질(턴 수, 사용자 발화 길이 대비 캐릭터 합산 길이 등 — v1.5의 MI Talk Time 원칙)
4. v1.5 4-캐릭터 체제에서 **요기·숨이 발화는 분석 제외** (필터링)
