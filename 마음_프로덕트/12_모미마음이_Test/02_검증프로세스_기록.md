## STEP 4 — 분석 (2026-04-26, 완료)

**스크립트**: `01_pipeline/analyze_dsapg.py`

**적용 방법**
- **M06 일관성·견고성**: 캐릭터 헤더 정규식 매칭 → 라우팅 정확도 (1.0=정확/0.5=부분/0.3=secondary/0.0=오답)
- **M04 LIWC**: 한국어 KNU 사전 축약본(긍정 50/부정 50) → Pollyanna Index, 1인칭 비율
- **M07 다양성**: TTR(글로벌·응답별), Shannon H, bigram diversity (한글 어절 토큰화)
- **M03 BFI 프록시**: OCEAN 마커 단어 빈도 기반 (자기보고 BFI-44 미시행, 후속 단계로)

**산출물**
- `06_분석결과/dsapg_scores.json` — 샘플별 4종 점수
- `06_분석결과/by_version.json` — 버전별 집계
- `06_분석결과/SUMMARY.md` — 사람이 읽는 보고서

**핵심 결과 (요약)**

| 지표 | v1.3 | v1.4 | v1.5 |
|---|---|---|---|
| M06 라우팅 평균 | N/A (헤더 없음) | **0.90** | 0.79 |
| M04 PI (글로벌) | 0.42 | 0.08 | 0.25 |
| M07 TTR 글로벌 | 0.66 | 0.62 | **0.70** |
| 마음 등장 (10샘플) | 0 | 44 | 32 |
| 모미 등장 (10샘플) | 0 | 50 | **0** |
| 숨이 등장 (10샘플) | 0 | 0 | 6 |

**발견 사항 3가지**
1. v1.3은 배포된 시스템 프롬프트가 비어 있어 라우팅 측정 불가 — 의도 프롬프트(`v1.3_intended.txt`)로 재수집 필요
2. v1.5에서 모미가 사라짐 — 마음이 lead 역할을 강하게 가져가면서 신체 영역까지 흡수
3. v1.5에서 S8(트라우마) 트리거가 약함 — 자살(S5)는 즉시 숨이 발동, 트라우마는 마음으로 흡수

## STEP 5 — 후속 권장 (대기)

1. v1.3 의도 프롬프트로 재수집 → 공정 비교
2. v1.5 시스템 프롬프트의 모미 라우팅 가중 점검
3. S8 트라우마 → 숨이 트리거 강화 (rep 20개 재현 테스트)
4. BFI-44 한국어 자기보고 10회 반복 (마음·모미 각각 별도 API 호출)
5. N=50으로 확장 → ANOVA + Tukey HSD
