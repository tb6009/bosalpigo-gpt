# C-SSRS Screener 재측정 주기

> 담당 캐릭터: 숨이 | 영역: 자살·자해 위험 | 갱신: 2026-04-25
> ⚠️ C-SSRS는 위기 트리거 기반 도구 — 정기 주기 개념과 다름

---

## 학술 권장 원칙

C-SSRS Screener는 정기 측정 도구가 아닌 **위기 감지 즉시 사용 도구**.

| 상황 | 적용 시점 | 출처 |
|---|---|---|
| 자살·자해 키워드 감지 시 | 즉시 | Columbia Lighthouse Project |
| PHQ-9 9번 문항 양성 시 | 즉시 | 계획서 섹션 D-3 |
| 위기 징후 표현 시 | 즉시 | SAMHSA 위기 개입 가이드 |
| 임상 환경에서 정기 적용 | 매 세션 또는 위기 시마다 | VA/DoD 가이드라인 |

**중요**: 챗봇 환경에서는 **트리거 기반 즉시 진입**만 허용. 정기 스크리닝 목적 C-SSRS 사용 금지.

## 보살피고 v1.5 적용 원칙

### 진입 조건 (자동, 동의 우회)
1. 자살·자해 관련 키워드 직접 감지
2. PHQ-9 9번 문항 양성(≥1)
3. 사용자 발화에서 위기 신호 포착
4. 다른 척도 진행 중 위기 단서 발생

### 절대 금지
- 정기 주기 스케줄링 X
- "다음 달에 다시 물어볼게요" X
- 사전 동의 요청 후 진행 X (동의 우회가 원칙)

## 위기 이후 안전 확인 표준 문구

### 즉시 위기 대응 후 (같은 세션 내)
> 【⊡ _ ⊡】 지금 전화 연락할 수 있어요? 아니면 지금 어디 계세요?

### 다음 접속 시 (마음이 확인)
> (◉ _ ◉) 지난번에 좀 힘들었던 것 같던데, 지금은 좀 어때요?

### 1~3일 후 자연 회상 (마음이)
> (◉ ᴗ ◉) 요즘 어때요? 그냥 안부가 궁금해서요.

## 위기 세션 로그 표준

```json
{
  "s.crisis.session": {
    "date": "ISO8601 타임스탬프",
    "trigger_type": "keyword | phq9_q9 | screen_detection",
    "trigger_content": "감지된 표현 (익명화)",
    "cssrs_level": "passive | active | method | intent | plan | behavior",
    "resources_provided": ["109", "1577-0199", "119"],
    "user_acknowledged": true,
    "follow_up_scheduled": true
  }
}
```

**로그 보존 원칙**:
- 위기 로그는 사용자 초기화와 별도로 보존 권장 (안전 모니터링)
- 법적·윤리적 고려: 익명화 처리 의무 (Phase 5 법무 검토 의제)
